Configuring Application

Please referrer to "Prerequisites" and the "Installation" sections of this documentary.

When you finished other sections of getting started documentation and installed source of sdf, you can start building your software.
But you still need to configure your application. This documentary will help you how to configure it.

  1. Open sdf source directory with your chosen ide.
  2. Locate to sdf.go file.
  3. If your application has dynamic content and requires authentication you need to create .envrc (or .env) file on the root directory of sdf.go file.
    1. Open the .env file that you've just created.
    2. Add SESSION_KEY="32bytesofsessionkey" to .env file.
    3. Locate sdf.go
    4. Find err = godotenv.Load(filepath.Join(dir, ".envrc")) line and change it to your .env file.
  4. If you've static files uncomment 19. line and change assets directory to your static directory.
    Or simply add core.RegisterStaticHandle("/localdir/", "publicdir") to sdf.go file.
  5. And you're complete!

Creating Hello World Example

If you've finished configuring you're ready to create your hello world application!

  1. Open sdf source directory with your chosen ide.
  2. Locate to handlers directory.
  3. Create a new .go file of your chosen name (this example uses hi.go)
  4. Open hi.go and add the boilerplate code bellow; 
    5. 
package handlers
import (
    "SDF/core"
    "fmt"
    "net/http"
)

func init() {
    core.RegisterHandle("/", exampleHandle, "GET")
}

func exampleHandle(writer http.ResponseWriter, request *http.Request) {
    _, err := fmt.Fprintf(writer, "Hello World!")
    core.CheckErrorNotPanic(err)
}
  5. Open your terminal and run $ go run sdf.go
  6. Locate to http://localhost:8000 in your favorite browser
  7. You're now ready to create your own application.
